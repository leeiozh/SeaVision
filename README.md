# SeaVision

Система оценки параметров морского волнения по данным навигационного радара в реальном времени.

---

## Принцип работы

Навигационный радар формирует последовательность радиолокационных изображений (РЛИ) поверхности моря. Каждое РЛИ представляет собой двумерный массив интенсивности рассеяния в полярных координатах `(AAP × ARDP)` пикселей, получаемый за один оборот антенны. При скорости вращения 25 об/мин период дискретизации составляет 2,4 с, максимально различимая частота (частота Найквиста) — F<sub>max</sub> = RPM/120 ≈ 0,208 Гц, минимальный период ≈ 4,8 с.

## Алгоритм обработки

Вычислительный конвейер состоит из двух фаз: предварительного анализа для идентификации волновых систем и оценки тока, и финального расчёта спектральных характеристик.

### Фаза 1 — накопление спектра

Из каждого кадра РЛИ вырезаются **NUM_AREA = 8** пространственных сегментов равномерно по азимуту. Каждый сегмент — квадрат `2·ASP × 2·ASP` пикселей с центром на расстоянии ADP пикселей от антенны; вырезка выполняется с билинейной интерполяцией для компенсации полярной геометрии РЛИ.

После накопления N_SHOTS = 256 последовательных кадров для каждого сегмента вычисляется трёхмерный спектр мощности методом Welch:

1. Пространственное преобразование Фурье (`fft2`) применяется к каждому кадру по осям `(x, y)`, что даёт набор 2D пространственных спектров.
2. Метод Welch по временно́й оси (ось 0) обеспечивает статистически устойчивую оценку спектра. Берётся положительная половина по частоте: результирующий массив имеет форму `(N_SHOTS//2, 2·K_NUM, 2·K_NUM)`.
3. Спектры всех NUM_AREA сегментов усредняются — это повышает статистическую надёжность оценки и подавляет пространственно-неоднородные помехи.

Итоговый трёхмерный спектр `spec_3d_corr(ω, kx, ky)` является основой для дальнейшего анализа.

### Фаза 2 — предварительный анализ и оценка течения

**Коррекция на ход судна.** Движение судна вызывает Доплеровский сдвиг дисперсионной кривой на величину `Δω = kx·Ux_ship + ky·Uy_ship`, где `Ux_ship = SOG·sin(COG)`, `Uy_ship = SOG·cos(COG)`. Векторная коррекция переносит каждую ячейку `(kx, ky)` спектра: `spec_3d_ship = apply_doppler_3d_vec(spec_3d_corr, Ux_ship, Uy_ship)`.

**Идентификация волновых систем.** Из скорректированного спектра вычисляется 1D частотный спектр `S(ω)` с MTF-поправкой `k⁻¹˙²` (компенсация радарной передаточной функции). Функция `find_freq_peaks` находит до трёх значимых пиков с минимальным расстоянием между ними. По каждому частотному пику `find_system_dirs` определяет преобладающее направление из двумерного направленного спектра `s_om_th(θ, ω)`. Результат — список `systems_draft` с характеристиками `{ω, Δω_lo, Δω_hi, θ}` для каждой идентифицированной системы.

**Оценка вектора кажущегося течения.** Поверхностное течение смещает ω-k портрет вдоль оси частот: `ω_obs = √(g·k) + kx·Ux + ky·Uy`. Применяется двухпроходный МНК:
- *Pass 1 (argmax, широкое окно)*: первичная оценка вектора `(Ux, Uy)` по положению энергетического максимума.
- *Pass 2 (centroid, узкое окно ±_SIGNAL_BAND)*: уточнение центроидным методом в окрестности оценки из Pass 1.

При наличии **двух и более** волновых систем используется `calc_current_multiwave`: МНК выполняется независимо по каждой системе в угловом конусе ±45° вокруг её направления, затем результаты объединяются с равными весами. Это позволяет разделить вклады разных систем и повысить точность при угловом расхождении систем > 45°. При коллинеарных системах или вырожденной матрице — автоматический переход на однофазный `calc_current_vector`.

Вектор `(Ux, Uy)` — кажущаяся скорость в географическом фрейме (Восток, Север), включающая как течение воды, так и скорость судна. Значения, превышающие физически обоснованный предел 3,0 м/с, ограничиваются (клиппируются).

### Фаза 3 — финальный спектр и разбиение на системы

**Доплеровская коррекция** `apply_doppler_3d_vec(spec_3d_corr, Ux, Uy)` переносит спектр в систему отсчёта, движущуюся со скоростью течения, компенсируя сдвиг для всех волновых систем одновременно. Результат — `spec_3d_fixed(ω, kx, ky)`.

**Разделение сигнал/шум.** Вычисляется ω-k портрет `port_fixed(ω, k)`. Полоса `±_SIGNAL_BAND = 10` бинов вокруг теоретической дисперсионной кривой `ω = √(g·k)` считается сигналом; остальное — шумом. После MTF-коррекции `k⁻¹˙²` вычисляется:
- `snr_tot` — интегральное отношение сигнал/шум в ω-k пространстве;
- 1D спектр `S(ω)` → пиковый период T<sub>p</sub>, средний период T<sub>m</sub>, дисперсия m₀;
- значительная высота волн: **H₁/₃ = 0,01·(SNR_A + SNR_B·√snr_tot)** [м].

Направленный спектр `S(θ, ω)` вычисляется из `spec_3d_fixed` угловым разложением с тем же критерием отбора сигнала (_SIGNAL_BAND), что обеспечивает согласованность оценок.

**Разбиение на системы** (`calc_partitions`): итеративный поиск пиков в `S(θ, ω)` с гашением окрестности пика после каждой итерации. Системы классифицируются по угловому расстоянию до направления ветра (±45°): ближайшая по периоду — ветровые волны, остальные — зыби. Высота каждой системы: `H_i = H₁/₃ · √(frac_i / Σfrac_j)`, что обеспечивает сохранение суммарной энергии.

### Усреднение и вывод

`Averager` хранит кольцевой буфер последних MEAN = 4 результатов. `get_mean()` усредняет волновые параметры и нормирует спектральные массивы в диапазон [0, 255]. Первый результат передаётся сразу, не дожидаясь заполнения буфера.

```
РЛИ (UDP / NC / BT8)
    ↓  BackData + Navi
Manager  [Input → Process → Output]
    ↓
Processor.update()
    ├─ NUM_AREA сегментов, билинейная интерполяция
    ├─ 3D Welch FFT → spec_3d_corr
    ├─ Коррекция на ход судна → spec_3d_ship
    ├─ find_freq_peaks → find_system_dirs → systems_draft
    ├─ calc_current_multiwave / calc_current_vector  [клип 3.0 м/с]
    ├─ apply_doppler_3d_vec → spec_3d_fixed
    ├─ MTF k⁻¹˙², SNR, Tp, Tm, H₁/₃
    ├─ calc_spec2d → S(θ,ω), peak_dir, mean_dir
    └─ calc_partitions → ветровые / зыбь 1 / зыбь 2
    ↓
Averager  [MEAN=4, нормировка → [0,255]]
    ↓
UdpOutputSink (1412 байт)  /  CSVOutputSink (4 файла)
```

## Флаги качества

Поле `quality` (байт 43 UDP-пакета) принимает значение **1 (GOOD)** только при одновременном выполнении всех четырёх условий:

| Критерий | Порог | Смысл |
|---|---|---|
| `snr_tot` | ≥ 5,0 | Достаточная мощность волнового сигнала в ω-k спектре |
| `σ_ring` (std РЛИ в кольце ADP ± ASP) | ≥ WIND_SIG_MIN | Достаточная неоднородность поверхности моря |
| T<sub>peak</sub> | ≥ 5,5 с | Волны разрешаются радаром надёжно |
| n_sys | ≥ 1 | Выделена хотя бы одна волновая система |

**Важно:** `quality = 0` (BAD) означает снижение доверия к результатам, но **не их ошибочность**. Данные передаются в обоих случаях. Типичные сценарии BAD:

- Слабый ветер → малые волны → σ<sub>ring</sub> ниже порога при физически реальном, но слабом волнении;
- Пиковый период ниже 5,5 с → очень короткие ветровые волны, плохо разрешаемые радаром;
- Низкий SNR при высоком уровне шума РЛИ (осадки, помехи).

При обработке рекомендуется анализировать данные BAD совместно с вспомогательными параметрами (`snr_tot`, `wspd`, `n_sys`), а не отбрасывать их автоматически.

## UDP-протокол выхода (v2.0)

Размер пакета **1412 байт**, little-endian, UDP unicast на порт 4000.

```
Заголовок (52 байта) | spec_1d (64 байта) | spec_2d 36×36 (1296 байт)
```

Структура заголовка:

| Байты | Поле | Кодирование | Примечание |
|---|---|---|---|
| 0 | type | = 5 | идентификатор пакета |
| 1 | pulse | 1/2/3 | SP / MP / LP |
| 2–3 | step_mm | м × 1000 | разрешение по дальности |
| 4–5 | rpm_x100 | об/мин × 100 | |
| 6–7 | swh_sum | м × 100 | H₁/₃ суммарная |
| 8–9 | t_p_sum | с × 100 | пиковый период |
| 10–11 | t_m_sum | с × 100 | средний период |
| 12–13 | dir_p_sum | ° (целые) | пиковое направление, **матем.** |
| 14–15 | dir_m_sum | ° (целые) | среднее направление, **матем.** |
| 16–17 | swh_win | м × 100 | ветровые волны |
| 18–19 | t_p_win | с × 100 | |
| 20–21 | dir_p_win | ° (целые) | **матем.** |
| 22–23 | swh_sw1 | м × 100 | зыбь 1 (короткопериодная) |
| 24–25 | t_p_sw1 | с × 100 | |
| 26–27 | dir_p_sw1 | ° (целые) | **матем.** |
| 28–29 | swh_sw2 | м × 100 | зыбь 2 (длиннопериодная) |
| 30–31 | t_p_sw2 | с × 100 | |
| 32–33 | dir_p_sw2 | ° (целые) | **матем.** |
| 34–35 | curr_speed | м/с × 100 | uint16, клип 3,0 м/с |
| 36–37 | curr_dir | ° (целые) | **компас** (0°=Север, CW) |
| 38–39 | wspd_x10 | м/с × 10 | |
| 40–41 | wind_dir | ° (целые) | **матем.** |
| 42 | n_sys | 0–3 | uint8 |
| 43 | quality | 0/1 | uint8 |
| 44–45 | algo_version | 1, 2, … | версия алгоритма |
| 46–51 | reserved | = 0 | зарезервировано |

**Конвенция направлений.** Поля `dir_p/m_sum`, `dir_p_win/sw1/sw2`, `wind_dir` — математическая система: 0° = Восток, 90° = Север, счёт против часовой стрелки. `curr_dir` — компасная: 0° = Север, 90° = Восток, по часовой. Перевод: `φ_компас = (90° − φ_мат) mod 360°`.

**Спектры.** `spec_1d[i]` (байты 52–115) — нормированный 1D спектр, бин `i` соответствует частоте `f_i = i · F_max / 63`. `spec_2d[i][j]` (байты 116–1411) — направленный спектр 36 × 36, row-major: строка `i` = направление `i × 10°` (матем.), столбец `j` = частота `j · F_max / 35`. Оба массива нормированы в [0, 255] по максимуму за последние MEAN расчётов.

Подробное описание: `udp_protocol.docx`.

### Пример чтения на Python

```python
import socket, struct
import numpy as np

HDR      = '<BBHHHHHHHHHHHHHHHHHHHHBBHHHH'
HDR_SIZE = struct.calcsize(HDR)          # 52 байта
PKT_SIZE = HDR_SIZE + 64 + 36 * 36      # 1412 байт
RPM      = 25
F_MAX    = RPM / 120.0                   # ≈ 0.208 Гц

sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
sock.bind(('0.0.0.0', 4000))
data, _ = sock.recvfrom(PKT_SIZE)

un = struct.unpack(HDR + '64B', data[:HDR_SIZE + 64])
assert un[0] == 5, 'неверный тип пакета'

swh       = un[4]  / 100.0   # H₁/₃ суммарная [м]
t_p       = un[5]  / 100.0   # пиковый период [с]
t_m       = un[6]  / 100.0   # средний период [с]
dir_p     = un[7]             # пиковое направление [°, матем.]
n_sys     = un[22]            # число волновых систем
quality   = un[23]            # 0=BAD, 1=GOOD
algo_ver  = un[24]            # версия алгоритма
curr_speed = un[18] / 100.0  # скорость течения [м/с]
curr_dir  = un[19]            # направление течения [°, компас]
wspd      = un[20] / 10.0    # скорость ветра [м/с]

spec_1d = np.array(un[28:28 + 64], dtype=np.float32)
spec_2d = np.frombuffer(data[HDR_SIZE + 64 : PKT_SIZE],
                        dtype=np.uint8).reshape(36, 36).astype(np.float32)

# Перевод направления волн в компасный курс
dir_p_compass = (90 - dir_p) % 360

# Частотная ось spec_1d
f_axis = np.linspace(0, F_MAX, 64)          # [Гц]
# Частотная ось spec_2d (по столбцам)
f_axis_2d = np.linspace(0, F_MAX, 36)       # [Гц]
# Угловая ось spec_2d (по строкам, матем. конвенция)
theta_axis = np.arange(36) * 10.0           # [°]
```

## Сборка исполняемого файла

### Локальная сборка (Windows)

На Windows-машине с установленными зависимостями:

```bat
pip install -r requirements-win.txt
pip install pyinstaller
pyinstaller seavision-win.spec
```

Результат — папка `dist\seavision\` со всем необходимым. Скопировать `config.ini` рядом с `seavision.exe`.

### Автоматическая сборка через GitHub Actions

Сборка выполняется автоматически на Windows Server через GitHub Actions и не требует Windows-машины разработчика.

1. Убедиться, что изменения запушены в `origin/main`.
2. Перейти на GitHub → **Actions** → **Build Windows EXE**.
3. Нажать **Run workflow** (или дождаться автоматического запуска при пуше тега `v*`).
4. После завершения (~3–5 мин) скачать артефакт **seavision-windows-x64** со страницы запуска.
5. Распаковать, положить `config.ini` рядом с `seavision.exe`.

```bat
seavision.exe config_ship.ini
```
